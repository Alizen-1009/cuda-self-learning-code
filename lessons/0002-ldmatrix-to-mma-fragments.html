从 ldmatrix 到 mma.sync fragment

拆掉 WMMA：给定任意 lane，手算它加载、计算、写回哪些矩阵元素。

这节课的唯一目标 (tangible win) 完成后，你能审查一段裸 PTX MMA kernel：从 shared-memory 地址公式出发，验证 ldmatrix.x4/x2.trans 是否把 A/B 放进 mma.sync.m16n8k16 要求的寄存器 fragment，并反推出每个 lane 应写回的 C/D 坐标。

你已经用 WMMA 完成了真机验证。WMMA 替你隐藏了下面整条链路；今天只做一件事：把这条链路逐段接起来。

shared memory二维 A/B tile
lane address部分线程提供行首地址
ldmatrix集体搬到分散寄存器
mma.sync按固定 fragment 解释寄存器

1. 先固定我们要喂的指令

mma.sync.aligned.m16n8k16.row.col.f32.f16.f16.f32
    {d0,d1,d2,d3}, {a0,a1,a2,a3}, {b0,b1}, {c0,c1,c2,c3};

A 是 16×16 row-major，B 是 16×8 col-major，C/D 是 16×8。每线程持有 A 的 8 个 half、B 的 4 个 half、C/D 的 4 个 float。注意 PTX 中的 a0..a3 是四个 32-bit 寄存器；官方 fragment 公式中的元素编号 a0..a7 是每个寄存器内低到高的八个 half，二者不要混名。[1]

2. 一组公式解释全部 lane

令 g = lane_id >> 2，t = lane_id & 3。32 个 lane 被看成 8 组，每组 4 个线程：g 选行/列组，t 选组内相邻的两个元素。

fragment本 lane 持有的矩阵坐标
A行是 g 与 g+8；列是 2t, 2t+1, 2t+8, 2t+9。两行 × 四列，共 8 个 half。
B列固定为 g；行是 2t, 2t+1, 2t+8, 2t+9。四行 × 一列，共 4 个 half。
C/D行是 g 与 g+8；列是 2t, 2t+1。两行 × 两列，共 4 个 float。
lane 0 

  

    
A · 16×16
8 个 half → RA0..RA3

    
B · 16×8
4 个 half → RB0..RB1

    
C/D · 16×8
4 个 float → 四个独立的 RC0..RC3

  






  先验算 lane 13
  g=3, t=1。所以 C/D 必须是行 {3,11}、列 {2,3} 的四个元素。以后看到写回公式，先用一个非边界 lane 做这种心算，最容易抓出转置和 +8 错位。




3. ldmatrix 为什么刚好对上


ldmatrix.sync.aligned.m8n8.xN.shared.b16 把一个或多个 8×8、16-bit 矩阵从 shared memory 集体装入 warp 寄存器。每个 8×8 矩阵需要 8 个行首地址；x4 时 lane 0–7、8–15、16–23、24–31 分别提供四个矩阵的地址。每四个 lane 共同接收一行，每 lane 得到相邻两个 half，正好一个 32-bit 寄存器。[2]



// A: 把 16×16 切成四个 8×8 象限
uint32_t a_addr = __cvta_generic_to_shared(
    &s_a[lane % 16][(lane / 16) * 8]);
ldmatrix.sync.aligned.m8n8.x4.shared.b16
    {RA0, RA1, RA2, RA3}, [a_addr];

// B: 把 16×8 切成上下两个 8×8；trans 生成 col-major fragment
uint32_t b_addr = __cvta_generic_to_shared(&s_b[lane % 16][0]);
ldmatrix.sync.aligned.m8n8.x2.trans.shared.b16
    {RB0, RB1}, [b_addr];




  
寄存器A 中对应的 8×8 象限 / B 中对应的行段

  
RA0A[0:8, 0:8]：上左

  
RA1A[8:16, 0:8]：下左

  
RA2A[0:8, 8:16]：上右

  
RA3A[8:16, 8:16]：下右

  
RB0 / RB1B[0:8, :] / B[8:16, :]；.trans 让输出寄存器匹配 B 的 col-major fragment。






  最常见的 Agent 幻觉
  “所有 32 个 lane 都各自提供一个有效地址”并不精确。地址提供者取决于 .x1/.x2/.x4：分别只需要 8/16/32 个行地址；但整条指令仍要求整个 warp 一致执行。不要把“参与执行”与“地址被消费”混为一谈。




4. 从 accumulator 公式直接写回


int row0 = block_m + lane / 4;
int row1 = row0 + 8;
int col  = block_n + (lane % 4) * 2;

// 每个 RC 寄存器打包横向相邻的两个 half 时：
store32(&C[row0][col], RC0);
store32(&C[row1][col], RC1);


如果累加器是 f32，则每 lane 有四个独立寄存器，坐标顺序是 (row0,col)、(row0,col+1)、(row1,col)、(row1,col+1)。这就是你审查 epilogue 的锚点：先从官方 C/D fragment 反推 store，不要从“希望输出长什么样”正向猜。

即时回忆 2：B 使用 ldmatrix.x2.trans 的核心原因是什么？

  

    
    
    
    
  

  
.trans 改变装入寄存器的片段组织，使 shared-memory 行数据匹配 B 的 col-major MMA fragment；它不保证自动消除所有 bank conflict。
