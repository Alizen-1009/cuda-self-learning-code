术语表

Tensor Core、CUTLASS/CuTe、Hopper/Blackwell、MoE 的规范词汇。每节课都遵循这里的定义。

架构标签： ADA / sm_89（指令代际地基） HOPPER / sm_90 · H20 BLACKWELL / sm_100 · B200 / B300

一、Tensor Core 指令

Fragment（寄存器片段）
一个逻辑矩阵 tile 按硬件规定分散到 warp 各 lane 寄存器后的局部份额。它不是连续内存，也通常不是完整一行/列；审查 MMA 代码时必须区分“逻辑矩阵坐标”和“本 lane 的寄存器编号”。[PTX ISA: Matrix Fragments]
ldmatrix
warp 集体从 shared memory 装入一个或多个小矩阵 fragment 的 PTX 指令。.x1/.x2/.x4 分别需要 8/16/32 个行首地址；.trans 改变装入寄存器的片段组织以匹配转置布局。[PTX ISA: ldmatrix]
Address Provider（行地址提供者）
在 ldmatrix 中，其地址操作数实际被用于指定矩阵行首的 lane。整 warp 都执行指令，但不代表每个 lane 的地址都被消费；必须按 .x1/.x2/.x4 区分。
Tensor Core
SM 内专用硬件单元，一条指令完成小矩阵的乘累加 D = A·B + C。关键点：它是 warp（乃至 warpgroup）协作的，不是单线程指令——操作数分布在多个线程的寄存器里。[NVIDIA CUDA C++ Guide]
MMA / mma.sync ADA
Warp 级同步 MMA 的 PTX 指令。一个 warp 的 32 线程共同完成一块 m16n8k16（fp16 常见）的乘累加，A/B/C/D 的 fragment 按固定规则分散在 32 线程寄存器中。是理解一切上层 tensor core 的地基。[PTX ISA]
WMMA (nvcuda::wmma)
CUDA C++ 层对 mma 的高级封装（16×16×16），用起来简单但控制力弱、性能上限低于手写 mma.sync。多用于教学与入门。
WGMMA / wgmma.mma_async HOPPER
Warpgroup 级异步 MMA。由一个 warpgroup（4 warps = 128 线程）发起，操作数可直接来自 shared memory，累加器在寄存器中；配合 fence/mbarrier 实现计算与访存重叠。Hopper 高性能 GEMM 的核心。[cutlass/wgmma 笔记 · Colfax]
TMMA / tcgen05.mma BLACKWELL
你说的"TMMA"对应 Blackwell 第五代 tensor core 的 tcgen05 指令族（"TMMA" 是非正式叫法）。累加器放在新的 Tensor Memory (TMEM)，支持 CTA-pair（跨 2 个 SM）的 MMA，原生支持 FP8/FP4 block-scaled。[PTX ISA · CUTLASS Blackwell 示例]
Warpgroup HOPPER
连续 4 个 warp（128 线程），是 wgmma 操作的基本单位。

二、内存与数据搬运

内存层次
Global (HBM) → L2 → Shared Memory (SMEM) / L1 → Register；Blackwell 另加 TMEM（tensor core 专用）。越靠近计算单元越快越小。优化的本质就是把数据尽量留在高层次、复用最大化。
Shared Memory 资源上限
sharedMemPerBlock 是单 block 的默认上限，sharedMemPerBlockOptin 是显式 opt-in 后的单 block 上限，sharedMemPerMultiprocessor 是一个 SM 上所有驻留 block 共同竞争的总预算。它们共同约束 occupancy。详见 CUDA Device Properties 速查。
Persisting L2 Cache
从设备 L2 中预留一部分容量，给标为 cudaAccessPropertyPersisting 的访问更高保留优先级。它不是锁定或预加载；最大可预留量由 persistingL2CacheMaxSize 给出。[CUDA C++ Guide: L2 Access Management]
Access Policy Window
附着在 CUDA stream 或 graph kernel node 上的连续地址区间，用 hitProp、missProp 和 hitRatio 提示 L2 缓存策略。窗口大小上限是 accessPolicyMaxWindowSize；它描述可标注的地址范围，不是缓存容量。
cp.async (LDGSTS) AMPERE+
异步拷贝 global→shared，绕过寄存器，让搬运与计算重叠。是 software pipelining 的基础。
Copy Engine / DMA Engine
独立于 SM 计算流水线执行数据搬运的硬件引擎；asyncEngineCount 报告其数量。配合 pinned host memory、cudaMemcpyAsync 与可并发 stream，拷贝才可能和 kernel 计算重叠。
Pinned / Mapped Host Memory
Pinned 表示 host 页被锁定，可供 DMA 高效搬运；mapped 在此基础上把 host memory 映射进 GPU 地址空间，让 kernel 可以直接 load/store。离散 GPU 上的 mapped 访问通常仍需经过 PCIe/NVLink。
TMA — Tensor Memory Accelerator HOPPER
专用硬件 DMA 引擎，用多维 tensor descriptor 一条指令（cp.async.bulk.tensor）搬运整块 tile，完成信号写入 mbarrier。大幅降低地址计算与线程开销。[cutlass/tma 笔记 · Colfax]
mbarrier
Shared memory 中的屏障对象，用于跟踪 TMA / wgmma 等异步操作的完成，是生产者-消费者流水线的同步原语。
Bank conflict
Shared memory 分 32 个 bank；同一 warp 的线程访问同一 bank 的不同地址会被串行化。避免它是 SMEM 优化的核心。
Swizzle
对 shared memory 地址做置换（XOR 等），使 tensor core 所需的访问模式恰好避开 bank conflict。CuTe 用 Swizzle<B,M,S> 描述。[repos/LeetCUDA/kernels/swizzle]
TMEM — Tensor Memory BLACKWELL
Blackwell 新增的片上内存空间，专门存放 tensor core 累加器，独立于寄存器与 shared memory。tcgen05 直接读写它。

三、CUTLASS / CuTe 抽象

CUTLASS
NVIDIA 开源的高性能 GEMM/卷积模板库。2.x 用 fragment/iterator 风格；3.x 基于 CuTe 重写。Agent 生成的高性能 kernel 常是 CUTLASS 风格。[github.com/NVIDIA/cutlass]
CuTe
CUTLASS 3.x 的核心 C++ 模板库，用统一的 Layout 代数描述数据与线程的排布。看懂 CuTe = 看懂现代 CUTLASS。[cutlass/cute 笔记]
Layout = (Shape, Stride)
CuTe 的核心概念：从逻辑坐标到线性偏移的映射，由形状与步长组成。Tensor = 指针 + Layout。
Atom / TiledMMA / TiledCopy
Atom 封装一条硬件指令（一个 mma.sync 或一个 ldmatrix）；TiledMMA/TiledCopy 把 Atom 按线程铺开成一个 tile 的完整操作。这是 CuTe 组织 tensor core 的方式。

四、性能与调度

Concurrent Kernels
设备同时执行来自不同 stream 的多个 kernel 的能力。concurrentKernels = true 只表示硬件支持；无依赖且仍有 SM、寄存器与共享内存资源时才会真正重叠。
Warp specialization HOPPER+
把一个 block 的 warp 分成生产者（用 TMA 搬数据）和消费者（用 wgmma 算），通过 mbarrier 流水衔接。Hopper/Blackwell 高性能 GEMM 的标准结构。
Software pipelining / 多级缓冲
预取若干 stage 的数据到 SMEM，使第 k 块的计算与第 k+1 块的搬运重叠，隐藏访存延迟。stage 数是关键调参。
Occupancy
SM 上活跃 warp 数 / 最大可容纳 warp 数。受寄存器数、SMEM 用量限制。高不一定好——tensor core kernel 常靠流水而非高 occupancy 藏延迟。
Roofline / Arithmetic Intensity
算术强度 = FLOP/Byte。低于 roofline 拐点 → 访存受限；高于 → 计算受限。判断"该优化访存还是算力"的第一工具。[cuda-mode L8]

五、MoE 计算侧（主线）

Grouped / Segmented GEMM
每个 expert 有独立权重、且分到的 token 数不等，于是把多个大小不一的 GEMM 打包成一次 kernel 调用。是 MoE 计算侧的核心。[cutlass/gemm · llm/moe 笔记]
Permute / Unpermute
按路由结果把 token 重排，使去往同一 expert 的 token 连续（gather），expert GEMM 后再还原（scatter）。
moe_align_block_size
把每个 expert 的 token 数对齐/padding 到 tile block 大小，让 grouped GEMM 能整块高效执行的辅助 kernel。[llm/moe/moe_align_block_size 笔记]
量化 GEMM (W4A16 / W8A8 / Mixed GEMM)
权重/激活用低精度（INT4/INT8/FP8）以省带宽与显存，tensor core 做混合精度乘累加。MoE 推理重度使用。[cutlass/gemm TRT-LLM 笔记]
FP8 (E4M3/E5M2) · FP4 / MXFP4 BLACKWELL
低精度浮点格式。Blackwell tensor core 原生支持 FP4 及 block-scaled / microscaling（每小块一个缩放因子）以保精度。
